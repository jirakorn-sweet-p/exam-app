# ระบบทำข้อสอบออนไลน์ (Exam App)

เว็บแอปทำข้อสอบที่ต้อง login (username/password) ก่อนเข้าใช้งาน พร้อมระบบให้คะแนนอัตโนมัติและเฉลย

- **Frontend:** HTML/CSS/JavaScript (เสิร์ฟจาก Express)
- **Backend:** Node.js + Express
- **Database:** PostgreSQL (Neon)
- **Auth:** bcrypt (hash รหัสผ่าน) + JWT เก็บใน httpOnly cookie
- **Deploy:** Render

---

## รันบนเครื่องตัวเอง (local)

> ระบบจะ **สร้างตารางและโหลดข้อสอบ 99 ข้อให้อัตโนมัติ** เมื่อเริ่มเซิร์ฟเวอร์ครั้งแรก ไม่ต้องรันคำสั่งเตรียมฐานข้อมูลเอง

1. ติดตั้ง dependencies
   ```bash
   npm install
   ```

2. สร้างไฟล์ `.env` จากตัวอย่าง แล้วใส่ค่า `DATABASE_URL` จาก Neon
   ```bash
   cp .env.example .env
   ```

3. เริ่มเซิร์ฟเวอร์ได้เลย
   ```bash
   npm run dev
   ```
   เปิด http://localhost:3000 — เข้าสู่ระบบด้วย **student01** / **123456** หรือกดสมัครสมาชิกใหม่

### (ทางเลือก) รันสคริปต์เตรียมข้อมูลเอง

ปกติไม่จำเป็น แต่ถ้าต้องการจัดการเองมีคำสั่งให้:
```bash
npm run init-db      # สร้างตารางอย่างเดียว
npm run seed-exam1   # โหลดข้อสอบวิชาสื่อสาร 99 ข้อ
npm run seed         # โหลดข้อสอบตัวอย่าง 5 ข้อ (ถ้าต้องการ)
```

## ตั้งค่า Neon (ฐานข้อมูล)

1. สมัคร https://neon.tech แล้วสร้าง Project ใหม่
2. ไปที่หน้า **Dashboard → Connect** คัดลอก connection string
   (รูปแบบ `postgresql://user:pass@host/db?sslmode=require`)
3. นำค่านั้นไปใส่เป็น `DATABASE_URL`

## Deploy บน Render

### วิธีที่ 1 — ผ่านหน้าเว็บ (แนะนำ)

1. push โค้ดนี้ขึ้น GitHub
2. ไปที่ Render → **New → Web Service** → เชื่อม repo
3. ตั้งค่า:
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
4. ใส่ Environment Variables:
   | Key | Value |
   |-----|-------|
   | `DATABASE_URL` | connection string จาก Neon |
   | `JWT_SECRET` | ข้อความสุ่มยาว ๆ |
   | `NODE_ENV` | `production` |
   | `PGSSL` | `true` |
5. กด **Create Web Service**

### วิธีที่ 2 — ใช้ Blueprint

มีไฟล์ `render.yaml` อยู่แล้ว เลือก **New → Blueprint** แล้วชี้ไปที่ repo
จากนั้นกรอกค่า `DATABASE_URL` เอง

### สร้างตารางบน production

ไม่ต้องทำเอง — เมื่อ Render สตาร์ทเซิร์ฟเวอร์ครั้งแรก ระบบจะสร้างตารางและโหลดข้อสอบให้อัตโนมัติ
(เห็นได้จาก log ว่า "ฐานข้อมูลพร้อมใช้งาน")

## หน้าผู้ดูแลระบบ (Admin)

เข้าที่ `/admin.html` (เช่น http://localhost:3000/admin.html) แล้วเข้าสู่ระบบด้วยบัญชี admin

- บัญชีเริ่มต้น: **admin / admin123** (เปลี่ยนได้ด้วยตัวแปร `ADMIN_PASSWORD` ก่อนสตาร์ทครั้งแรก)
- เมื่อเข้าสู่ระบบด้วยบัญชี admin หน้า dashboard ผู้สอบจะมีปุ่ม "แผงผู้ดูแล" ให้กดเข้าด้วย

สิ่งที่ดูได้ในแผงผู้ดูแล:
- **ภาพรวม** — จำนวนผู้สอบ, ชุดข้อสอบ, จำนวนครั้งที่สอบ, คะแนนเฉลี่ยรวม
- **ผู้ใช้** — รายชื่อผู้ใช้ทั้งหมด พร้อมจำนวนครั้งที่สอบ คะแนนเฉลี่ย และเวลาที่สอบล่าสุด มีปุ่ม "ดูคำตอบ" เพื่อดูการสอบทั้งหมดของคนนั้นและเข้าไปดูว่าตอบข้อไหนว่าอย่างไร (เทียบกับเฉลย)
- **ประวัติการสอบ** — การส่งคำตอบทุกครั้ง กรองตามข้อสอบได้ คลิกดูคำตอบรายข้อเทียบเฉลย และดาวน์โหลดเป็น CSV
- **วิเคราะห์ข้อสอบ** — อัตราการตอบถูกของแต่ละข้อ เรียงข้อที่ตอบผิดมากขึ้นก่อน (ดูว่าข้อไหนยากหรือควรทบทวน)
- **ข้อสอบ (ดาวน์โหลด)** — ดาวน์โหลดข้อสอบเป็น PDF ขาวดำสำหรับพิมพ์ทำข้อสอบ มี 2 แบบ: "PDF (ทำข้อสอบ)" ไม่มีเฉลย และ "PDF + เฉลย" แนบหน้าเฉลยท้ายเล่มสำหรับผู้ตรวจ (ภาษาไทยเรนเดอร์ด้วยฟอนต์ Sarabun, รูปแปลงเป็นขาวดำ และลบสีที่บอกเฉลยออกแล้ว)

## การทำข้อสอบ (ฝั่งผู้สอบ)
- **สลับลำดับข้อ** — ทุกครั้งที่เริ่มทำ ระบบจะสุ่มสลับลำดับข้อให้อัตโนมัติ (การให้คะแนนอ้างอิงด้วย id ของข้อ จึงไม่กระทบความถูกต้อง)
- **สอบซ้ำได้ไม่จำกัด** — กด "เริ่มทำ" วิชาเดิมได้เรื่อย ๆ ระบบบันทึกผลทุกครั้งแยกกัน
- **ประวัติทุกรอบ** — หน้าแรกของผู้สอบแสดงผลทุกครั้งพร้อม "ครั้งที่" และทำเครื่องหมาย "สูงสุด" ที่รอบคะแนนดีที่สุดของแต่ละวิชา

API ฝั่ง admin (ต้องเป็น role `admin` เท่านั้น):

| Method | Path | คำอธิบาย |
|--------|------|----------|
| GET | `/api/admin/stats` | สถิติภาพรวม |
| GET | `/api/admin/users` | รายชื่อผู้ใช้ |
| GET | `/api/admin/attempts` | ประวัติการสอบ (กรอง `?exam_id=`) |
| GET | `/api/admin/attempts/:id` | รายละเอียดการสอบรายข้อ |
| GET | `/api/admin/exams` | รายการข้อสอบพร้อมสถิติ |
| GET | `/api/admin/exams/:id/analysis` | วิเคราะห์รายข้อ |
| GET | `/api/admin/exams/:id/pdf` | ดาวน์โหลดข้อสอบเป็น PDF (`?withKey=1` แนบเฉลย) |

> ความปลอดภัย: ควรตั้ง `ADMIN_PASSWORD` เป็นค่าที่เดายากบน production และเปลี่ยนรหัสผ่าน admin หลังใช้งานครั้งแรก

```js

  choices: ["ตัวเลือก ก", "ตัวเลือก ข", "ตัวเลือก ค", "ตัวเลือก ง"],
  correct_index: 1   // 0 = ข้อแรก
}
```

## ข้อสอบในระบบ (โหลดอัตโนมัติ)

ระบบจะโหลดข้อสอบทุกวิชาให้เองตอนสตาร์ทครั้งแรก (ไฟล์อยู่ใน `db/`):

| วิชา | ไฟล์ | จำนวนข้อ | รูป |
|------|------|---------|-----|
| สื่อสาร | `exam1.json` | 99 | 30 |
| อาวุธ | `exam_weapon.json` | 100 | 2 |
| ยานยนต์ | `exam_vehicle.json` | 100 | 0 |
| ยุทธวิธี | `exam_tactic.json` | 100 | 4 |

- เฉลยทุกวิชาตรวจสอบกับ "ตัวอักษรสีแดง" ในไฟล์ PDF ต้นฉบับ
- รูปประกอบเก็บใน `public/exam-images/` (ในรูปที่มาจากการตัดหน้า PDF ได้ลบสีแดงที่บอกเฉลยออกแล้ว)
- การเพิ่มวิชาใหม่ภายหลัง: วางไฟล์ JSON ใน `db/` แล้วเพิ่มชื่อไฟล์ในอาเรย์ `EXAM_FILES` ใน `db/bootstrap.js` ระบบจะโหลดวิชาที่ยังไม่มีให้เองตอนสตาร์ต (วิชาที่มีอยู่แล้วจะไม่ซ้ำ)

โครงสร้างแต่ละข้อในไฟล์ JSON:
```json
{
  "n": 1,
  "body": "โจทย์คำถาม ...",
  "choices": ["ตัวเลือก ก", "ตัวเลือก ข", "ตัวเลือก ค", "ตัวเลือก ง"],
  "correct": 3,
  "image": "exam-images/xxx.png"
}
```
`correct` คือ index ของคำตอบ (ก=0, ข=1, ค=2, ง=3) และ `image` ใส่เฉพาะข้อที่มีรูป

## ข้อสอบวิชาสื่อสาร (99 ข้อ)

ไฟล์ `db/exam1.json` คือข้อสอบวิชาสื่อสาร 99 ข้อ ที่ดึงจากไฟล์ PDF ต้นฉบับ
รวมโจทย์ ตัวเลือก เฉลย และรูปภาพประกอบ (ข้อ 39–69 เก็บไว้ใน `public/exam-images/`)

- เฉลยทุกข้อตรวจสอบกับ "ตัวอักษรสีแดง" ในไฟล์ PDF ต้นฉบับ
- ข้อ 60 ไม่มีในไฟล์ต้นฉบับ (เลขข้อข้ามจาก 59 ไป 61) จึงมีข้อสอบจริง 99 ข้อ
- โหลดเข้าระบบด้วย `npm run seed-exam1`

แต่ละข้อในไฟล์ JSON มีรูปแบบ:
```json
{
  "n": 62,
  "body": "จากรูป ...",
  "choices": ["...", "...", "...", "..."],
  "correct": 3,
  "image": "exam-images/q62.png"

```
ฟิลด์ `correct` คือ index ของคำตอบที่ถูก (ก=0, ข=1, ค=2, ง=3) และ `image` ใส่เฉพาะข้อที่มีรูป

## หมายเหตุด้านความปลอดภัย

- รหัสผ่าน hash ด้วย bcrypt ไม่เก็บเป็น plain text
- token เก็บใน httpOnly cookie (JS ฝั่ง client อ่านไม่ได้ ลดความเสี่ยง XSS)
- เฉลย (`correct_index`) ไม่ถูกส่งไปฝั่ง client ตอนทำข้อสอบ — ตรวจคะแนนที่ backend
- ตั้ง `JWT_SECRET` ให้สุ่มและเดายากเสมอใน production
